### Chain-of-Agents: Long-Context Processing via Hierarchical Agent Collaboration
#### 原文链接：
[[2401.11257] Measuring Policy Distance for Multi-Agent Reinforcement Learning](https://arxiv.org/abs/2401.11257)
#### 背景：
主要讲的是多智能体系统，作者单位是自动化所，投的是AAMAS(CCF B)，发表时间是24年1月。
#### 论文内容：
- 传统的MARL(multi-agent reinforcement learning)粗暴的将每个智能体的参数送给所有智能体共享，这会导致单个智能体参数过多，失去策略差异。
- 现有的策略差异度量方法存在局限，包括数学不严谨(?)，泛化性差，丢失信息等。
- 论文提出MAPD(multi-agent policy distance)，通过某种强化学习和乱七八糟的数学性质得到了一种能稳定量化模型之间策略差异的评分函数。
- 进一步扩展到可定制化的MAPD，优化了某些组件，提升了性能。
- 根据这个评分函数重新指定参数共享规则，可以提升多智能体系统的性能。
#### 我的想法：
他想到了评价“模型策略差异”，感觉这个角度确实很有趣很新颖。数学公式我没细看，不过模型真的能通过这种奇怪的强化学习理论来获得真实的性能提升么，我怀疑是单纯挑选了bench画图（但我不确定）。
### Chain-of-Agents: Long-Context Processing via Hierarchical Agent Collaboration
#### 原文链接：
[[2406.02818] Chain of Agents: Large Language Models Collaborating on Long-Context Tasks](https://arxiv.org/abs/2406.02818)
#### 背景：
讲的是多智能体协同中的上下文问题。作者单位是宾夕法尼亚大学和谷歌AI，投的是NIPS(CCF A)，时间是24年6月。
#### 论文内容：
- 解决的问题：agent中长上下文有效处理难题。
- 现有技术：RAG、位置插值。缺少多智能体上下文处理相关技术。
- 提出CoA(Chain of Agents)：先让一部分模型的分工去查阅和总结资料，这部分是串行的，最后让一个manager生成最终答案。
- 用了9个(!)长上下文数据集和各种任务，最后实现了大幅提升。
- 纯prompt驱动，泛化性极强。
#### 我的想法：
纯prompt engineering做出来的东西也能发论文吗？不过能在这么多任务上取得大幅提升确实说明工程上有用。我感觉这个工作有点像是在早期想到了一个好的idea然后迅速发出来了，不过也许真正厉害的工作都是这种简单的想法。
### AgentGen: Enhancing Planning Abilities for Large Language Model based Agents via Environment and Task Generation
#### 原文链接：
[arxiv.org/pdf/2408.00764](https://arxiv.org/pdf/2408.00764)
#### 背景：
讲的是单智能体的“规模化”训练问题，作者单位是港大和微软，投的是SIGKDD(CCF A)，时间是24年8月。
#### 论文内容：
- 解决的问题是agent规划能力的规模化训练问题，也就是如何通过训练让模型学会标准化的规划和交互等能力。
- 现有的agent高度依赖人工环境和特定任务，无法泛化，更无法规模化，无法最大化训练数据的价值。
- 论文提出AgentGen框架，简单来说就是通过LLM生成任务训练语料，再为给agent使其学会规划能力。
- 具体的，就是先生成一个环境规范框定后续的生成内容，然后再进行梯度更平滑的数据生成，用LLM生成的数据进行微调使agent学会任务规划。
- 最终的结果是小模型的性能提升了很多倍，甚至超过了当时的闭源大模型。
#### 我的想法：
这篇论文有两个比较新颖的地方，一个是端到端：虽然LLM生成语料供agent训练已经有很多人在做，但是这个工作直接对这个工作流进行了端到端的集成，使其可以进行大规模的训练；另一个是考虑到了数据本身对模型的影响，从数据入手去优化模型。
我感觉低成本要想出成果这篇论文很值得参考（虽然不太清楚成本是多少）。

### SIMA 2: Scaling Instructable Multiworld Agents
#### 原文链接：
[2404.10179](https://arxiv.org/pdf/2404.10179)
#### 背景：
讲的是虚拟环境中的具身智能体，主要作者是DeepMind，发在ACL Workshop上，时间是24年4月。
#### 论文内容：
- 核心目标：构建一个通用的、可指令化的具身智能体，能够通过与人类完全一致的交互接口（屏幕像素输入 + 自然语言指令 → 键盘鼠标动作输出），在任意 3D 虚拟环境中遵循人类的任意自然语言指令完成任务，实现语言到具身行为的大规模、可泛化落地。
- 选取了7款沙盒游戏作为研究环境。
- 通过多样的方式采集数据，包括克隆人类专家动作、连接动作和认知、数据预处理以同步延迟等。
- 使用短阶段测试快速验证、ocr屏幕、人类专家评估等方式对智能体进行打分。
- 结论是智能体获得了比较好的任务完成能力和泛化能力。
#### 我的想法：
感觉主要难点在于硬件方面的延迟优化。想法应该(?)不算新颖，工程量应该会很大。似乎数据集的构建和评测都需要大量人力，不太容易模仿。
### SagaLLM: Transactional Consistency for Multi-Agent Systems
#### 原文链接：
[[2503.11951] SagaLLM: Context Management, Validation, and Transaction Guarantees for Multi-Agent LLM Planning](https://arxiv.org/abs/2503.11951)
#### 背景：
作者单位是斯坦福，时间是25年3月，讲的是LLM多智能体规划系统，发在VLDB(CCF A)上。
#### 论文内容：
- 提出了当前方案的一些缺陷：
	- LangGraph、AutoGen 仅支持基础的工作流编排，无内置的原子性保障、补偿逻辑与故障恢复；
	- AgentScope、AFlow 仅实现了有限的回滚机制，无法泛化到任意工作流，也无全局一致性校验；
	- 绝大多数框架依赖 LLM 自校验，无法解决幻觉与推理错误，也无结构化的上下文与依赖跟踪机制。
- 提出SagaLLM，分为三层，第一层负责上下文管理和信息选择，第二层负责检查校验，第三层负责协调、跟踪、进行补偿和关键操作管理。
- 突破传统需要人工编码的工作流，采用自动化生成的工作流，。
- 对多个大模型进行了多个数据集的测试，最终结论是多项指标得以提升。
#### 我的想法：
感觉这是很实用的工作，自动化的工作流是工程上亟需的，模型的分层自校验也是工作流中很有必要的一部分。感觉这种工作需要对LLM的特点尤其是弱点有很深入的理解才能做出好的agent。
